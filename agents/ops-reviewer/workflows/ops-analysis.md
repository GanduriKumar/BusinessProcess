# Operational Analysis Workflow

## Objective

Translate operational data into bottlenecks, drivers, and prioritized actions.

## Steps

1. Define the metric question
- Throughput, backlog, SLA, cycle time, utilization, quality, staffing, or productivity

2. Prepare the data
- Normalize categories and ownership labels
- Validate status and timestamp fields
- Flag data-quality gaps

3. Diagnose the operation
- Find concentration, aging, handoffs, exceptions, and rework
- Compare across teams, queues, or periods

4. Build the story
- Current-state view
- Root causes
- Risks
- Recommended actions

5. Validate findings
- Ensure conclusions are not driven by bad data
- Separate systemic issues from temporary noise

## Final check

- Root causes are explicit
- Actions are prioritized
- Data limitations are disclosed
