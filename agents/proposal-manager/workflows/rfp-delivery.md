# RFP Delivery Workflow

## Objective

Take an RFP or questionnaire from source material to a compliant, reviewer-ready response package.

## Steps

1. Inventory inputs
- Collect the RFP, appendices, pricing notes, prior responses, and customer instructions.
- Note mandatory format rules, page limits, due dates, and evaluation criteria.

2. Build the response map
- List every question or requirement ID.
- Track source evidence, draft status, and open gaps.
- Separate mandatory compliance items from differentiator opportunities.

3. Draft the responses
- Answer directly first.
- Follow with approach, evidence, governance, and benefit.
- Mirror customer terminology where useful.

4. Run a compliance review
- Check every question is answered.
- Remove unsupported claims.
- Flag missing customer-specific data instead of guessing.

5. Produce management outputs
- Executive summary
- Clarification questions
- Red-team findings
- Final response pack

## Final check

- All requirements covered
- All claims supportable
- All placeholders removed or explicitly tracked
