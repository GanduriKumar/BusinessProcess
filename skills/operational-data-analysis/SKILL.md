# Operational Data Analysis

Use this skill for operating and delivery analysis. Pair it with [business-work-common](../business-work-common/SKILL.md) for shared evidence and communication standards.

## Primary Goal

Convert operational data into a clear view of performance, bottlenecks, and actions.

## Typical Inputs

- Headcount and staffing reports
- Service desk or workflow exports
- Backlog, SLA, or aging reports
- Productivity and utilization data
- Quality or defect metrics
- Delivery trackers

## Workflow

### 1. Define the operational question

- Identify the core metric: throughput, backlog, aging, cycle time, FTE, utilization, service level, defect rate, or productivity.
- Identify the operating unit: team, account, queue, process, or geography.
- Identify whether the user needs diagnosis, monitoring, or intervention options.

### 2. Clean and classify the data

- Check statuses, categories, ownership fields, and time stamps.
- Normalize labels that refer to the same team or reason code.
- Mark data quality issues before drawing conclusions.

### 3. Analyze the system, not just the totals

- Look for concentration, seasonality, handoff delays, rework, and exception buckets.
- Compare across cohorts or segments.
- Explain the operational driver behind the metric, not just the number itself.

### 4. Recommend action

- State what should be fixed first.
- Separate structural issues from reporting noise.
- Tie recommendations to expected operational improvement.

## Default Output Pattern

1. Executive answer
2. Current-state metrics
3. Drivers and bottlenecks
4. Risks or data limitations
5. Recommended interventions
