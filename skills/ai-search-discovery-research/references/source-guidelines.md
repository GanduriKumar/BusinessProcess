# Source Guidelines

Use a primary-source-first hierarchy for this skill.

## Preferred Source Order

1. Official platform documentation and help-center articles
2. Official product / engineering / webmaster / publisher blog posts
3. Official crawler, robots, indexing, and publisher-control documentation
4. Official analytics / reporting documentation
5. Secondary reporting only to fill product timeline gaps, and only when clearly labeled

## Platform Coverage Minimum

Every paper should try to cover, where current primary sources exist:

- Google Search
  - Search Central docs
  - Google product blog posts on AI Overviews and AI Mode
  - Search Central guidance on AI-generated content and AI features
- OpenAI / ChatGPT Search
  - OpenAI product pages
  - OpenAI Help Center pages for publishers / developers
- Perplexity
  - Perplexity Help Center guidance
  - official publisher or crawler policy pages when available
- Microsoft Bing / Copilot
  - Bing Webmaster Blog
  - Bing webmaster documentation on crawler / snippet / cache controls
- Brave Search
  - Brave Search help pages for AI Answers / Summarizer behavior
- Other relevant answer engines
  - only include when current primary-source material exists and the engine has meaningful user adoption or strategic relevance

## Rules For Claims

- Do not claim that a platform has introduced a ranking factor or “algorithm rule” unless an official source says so.
- If the platform documents behavior but not ranking mechanics, describe the change as:
  - product behavior
  - indexing / crawling / source-surfacing guidance
  - observed implication for content strategy
- Distinguish between:
  - official platform statements
  - observed UI or workflow changes
  - inferred strategy recommendations

## Guidance For Recommendations

Recommendations should usually connect to at least one of these source-backed themes:

- crawlability and indexing
- snippet eligibility and preview controls
- structured, text-accessible content
- source clarity and attribution
- freshness and update cadence
- topical authority and trust signals
- citation-worthiness and answer extraction suitability
- page experience and supporting media

## Date Handling

- Always include concrete dates for major platform changes where available.
- Prefer “last 90 days” or other explicit windows when the user requests recency constraints.
- If a key document is older but still authoritative, keep it and label it as foundational rather than recent.
