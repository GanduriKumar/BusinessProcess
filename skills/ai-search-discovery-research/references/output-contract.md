# Output Contract

The default deliverable is a markdown technical paper that is clear enough to cite internally and practical enough to drive an action plan.

## Required Sections

1. Executive summary
2. Scope and date of research
3. Search transition overview
4. Google Search
   - classic SEO / SEM assumptions that still matter
   - AI Overviews
   - AI Mode
   - what changed in discovery behavior
   - what Google explicitly says did not require special optimization
5. Per-engine sections for other relevant answer engines
   - OpenAI / ChatGPT Search
   - Perplexity
   - Microsoft Bing / Copilot
   - Brave Search
   - other currently relevant engines where primary sources exist
6. Implications for content design
   - blogs and editorial pages
   - technical documentation
   - Medium
   - LinkedIn
7. Content architecture recommendations
8. Measurement and operational checks
9. Concrete action plan
10. Source appendix

## Quality Bar

- Use current primary sources wherever possible.
- Use explicit dates for product launches and documentation updates when available.
- Make engine-by-engine differences easy to scan.
- State where advice is directly documented versus inferred from platform behavior.
- Recommendations must be specific enough for teams to implement.

## Minimum Recommendation Areas

The action plan should usually address:

- crawl / indexing controls
- page structure and information hierarchy
- answer extraction readiness
- attribution and trust signals
- author identity and source transparency
- content freshness and update patterns
- supporting visuals / structured data where relevant
- cross-posting strategy for owned sites, Medium, and LinkedIn

## Forbidden Failure Modes

- vague “write better content” advice
- unsupported claims that SEO is dead
- unsupported claims that one exact markup pattern guarantees AI visibility
- mixing official platform guidance with unsourced folklore
